2.PENYEBAB INKONSISTENSI
DILIHAT DARI POIN NOMOR 1, SELAMA MENJALANKAN 5X INI HASILNYA BERBEDA-BEDA KARENA RACE CONDITION, RACE CONDITION TERJADI KETIKA DUA THREAD ATAU LEBIIH MENGAKSES DAN MEMODIFIKASI VARIABEL YANG SAMA SECARA BERSAMAAN TANPA MEKANISME PENGAMANAN (LOCK). UNTUK MENGHINDARI KONDISI RACE CONDITION BISA MENGGUNAKAN "LOCK" PADA MODUL THREADING. CONTOHNYA TAMBAH VARIABEL "LOCK = THREADING.LOCK()" DAN GUNAKAN VARIABEL TERSEBUT SETELAH TIME.SLEEP DENGAN "WITH LOCK: COUNTER = TEMP + 1"
3. SKENARIO INTERLEAVING (CONTOH BUKTI)
SAYA SEDIKIT MENGUBAH RANGE NYA MENJADI 100, DAN BEGINI CONTOH SKENARIO RACE CONDITION:
	1. THREAD 1 MEMBACA COUNTER 100 = NILAI COUNTER 100
	2. THREAD 2 MEMBACA COUNTER 100 = NILAI COUNTER 100
	3. THREAD 1 MENGHITUNG COUNTER 100 + 1 = NILAI COUNTER 101
	4. THREAD 2 MENGHITUNG COUNTER 100 + 1 = NILAI COUNTER 101
	5. NILAI AKHIR COUNTER YAITU 101
KENAPA NILAI AKHIR COUNTER 101 DIKARENAKAN PENGHITUNGAN ANTARA THREAD 1 DAN 2 BERSAMAAN SEHINGGA NILAI AKHIRNYA 101, AKAN TETAPI APABILA KITA MENGGUNAKAN LOCK (POIN 2), MAKA SYSTEM AKAN MENGAKSES SATU THREAD DAHULU SAMPAI SELESAI BARU MENJALANKAN THREAD LAINYA.
